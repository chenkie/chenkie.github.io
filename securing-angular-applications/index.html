Building a robust application can be hard enough. Properly locking it down can be even harder. Any way you crunch it, getting authentication and authorization right is crucial for non-trivial apps.

What's so hard exactly?

Authentication and authorization in single page apps is a lot different than it is in traditional round-trip appliciations.

Having a separate client and server means you need to manage sessions on the front end while protecting data resources at the API in a stateless manner.

You also probably want to support social authentication through OAuth.

Focus on building your Angular app instead of trying to figure out authentication.

Learning about and implementing authentication and authorization in an Angular app can be hugely time-consuming. Not only do you need to search the web to learn about a ton of different concepts, you also need make sure that your implementation is rock solid.

Your time is valuable and it’s best-spent working on what matters: your actual application. Building in secure user authentication is a necessity, but it shouldn’t take as much time as building the rest of your app.
